第一周：文件系统侦探（FS Inspector）
Time: 2026-01-08 ~ 2026-01-14
主题：文件系统操作、路径处理、元数据、递归算法、结构化输出。
项目：rfind - 一个更友好、功能更强的find命令替代品。
核心任务：

递归遍历目录，支持基于名称、类型、大小的过滤。

显示文件的详细元数据（权限、修改时间、大小）。

计算目录总大小。

输出格式化的JSON或表格（使用tabled库）。
学习目标：

掌握 std::fs、std::path。

熟练使用 walkdir 库进行高效遍历。

使用 chrono 处理时间，用 serde_json 输出结构化数据。

编写安全的递归逻辑，避免符号链接循环。
扩展挑战：实现实时监控模式（使用 notify 库），当文件变化时发出通知。
技术栈：
    clap, // command工具
    walkdir,
    chrono, // 时间
    serde_json, // json
    tabled, // 表格或json
    notify, // 监控文件变化


第二周：文本处理大师（Text Processing Pipeline）
Time: 2026-01-15 ~ 2026-01-21
主题：流式处理、正则表达式、并发、命令行管道。
项目：rgrep + rsed - 一个支持高级搜索和替换的文本处理工具组合。
核心任务：

实现 rgrep：支持正则表达式、上下文显示、彩色输出、多种编码文件。

实现 rsed：支持基础的查找/替换，可读取管道输入。

让两个工具能通过管道协同工作（如 rgrep -n "error" log.txt | rsed "s/error/CRITICAL/g"）。
学习目标：

深度使用 regex 库。

掌握 clap 库构建复杂的命令行参数解析。

理解 std::io::Read/Write trait，实现管道兼容。

使用 rayon 库实现并行文件搜索，感受零成本抽象的并发。

学习 encoding_rs 处理不同编码。
扩展挑战：为常见日志格式（如JSON日志、Nginx日志）提供内置解析和字段提取功能。


第三周：进程与系统监视器（Process & System Monitor）
Time: 2026-01-22 ~ 2026-01-28
主题：进程管理、系统信息、跨平台抽象、实时UI。
项目：rtop - 一个跨平台的类 top / htop 系统资源监视器。
核心任务：

列出所有进程及其PID、状态、CPU/内存占用。

显示系统整体的CPU、内存、交换分区使用率。

实现基本的进程管理功能（发送信号、结束进程）。

提供刷新的终端界面（TUI）。
学习目标：

使用 sysinfo 库获取跨平台的系统和进程信息（避免直接调用平台API的复杂度）。

学习 crossterm 或 ratatui 构建终端用户界面。

理解进程和系统调用的基本概念。

初步接触 signal-hook 处理信号。
扩展挑战：实现进程树视图，或按网络IO对进程进行排序。


第四周：网络瑞士军刀（Network Swiss Army Knife）
Time: 2026-01-29 ~ 2026-02-04
主题：网络编程、TCP/UDP、简单协议、异步。
项目：nettool - 集成多种网络诊断功能的工具。
核心任务：

端口扫描器：简单的TCP连接扫描。

简易HTTP客户端：发送GET/POST请求，显示头部和响应（使用 reqwest 的阻塞模式入门）。

TCP Echo服务器/客户端：用于测试网络连通性。

DNS查询器：解析域名到IP。
学习目标：

掌握 std::net 进行同步网络编程。

理解TCP/UDP套接字的基本操作。

使用 reqwest 进行HTTP请求。

接触 async-std 或 tokio，将Echo服务器改为异步（可选，作为广度拓展）。

使用 trust-dns-resolver 进行DNS查询。
扩展挑战：实现一个简单的网络抓包器或流量统计工具（使用 pcap 或 libpnet）。



贯穿始终的工程实践
代码组织：从第一周就用Cargo workspace管理这四个工具，共享公共工具函数（如错误处理、日志）。

错误处理：统一使用 anyhow + thiserror，提供友好的错误信息。

测试：为每个工具的核心逻辑（如文件过滤算法、正则搜索）编写单元测试和集成测试。

文档：使用 cargo doc 生成文档，并为每个工具编写清晰的 --help 信息和README。

性能：在关键路径（如遍历、搜索）使用 std::time 或 criterion 进行简单的性能评估和优化。

如何执行
每日：安排1-2小时专注编码，周末可集中时间突破难点。

记录：为每个项目创建GitHub仓库，用Issues记录想法和Bug，用PR提交功能，模拟协作流程。
